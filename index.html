OVERVIEW
Character Coding maps characters to binary bit patterns.

ASCII (7-bit): 128 characters (0–127). Used in most English text systems.
ASCII (8-bit / Extended): 256 characters including special symbols.
EBCDIC: IBM's 8-bit encoding, used in mainframe systems.
Unicode UTF-8: Variable-width (1–4 bytes). Supports all world languages.

Parity Bits: An extra bit added to detect single-bit errors. Odd parity ensures the total number of 1s is odd; Even parity ensures it's even.

OVERVIEW
Pulse Code Modulation (PCM) converts an analog signal to digital via 3 steps:

1. Sampling: The analog signal is measured at regular intervals (Fs). Nyquist theorem requires Fs ≥ 2×f_max.
2. Quantization: Each sample is rounded to the nearest level. Levels = 2^n where n = bit depth. Causes quantization noise.
3. Encoding: Each quantized level is represented as a binary code word.

SNR Formula: SNR ≈ 6.02n + 1.76 dB (n = bit depth)

LINE CODING SCHEMES
NRZ-L: Level directly corresponds to bit value. Simple but has DC component.
NRZ-I: Transition at 1, no transition at 0. Better clock recovery.
RZ: Returns to zero halfway through each bit period.
Manchester: Always a mid-bit transition. Self-clocking.
AMI: Alternating Mark Inversion — zeros stay at 0, ones alternate +/-.
B8ZS/HDB3: Substitution codes that break up long zero runs.

OVERVIEW
ASK varies the amplitude of a carrier wave based on the digital message.

For bit 1 (Mark): amplitude = A1
For bit 0 (Space): amplitude = A0 (often 0 = OOK, On-Off Keying)

Demodulation: Envelope detection — rectify the signal, pass through LPF, then threshold compare.

Weakness: Very susceptible to amplitude noise.

QPSK maps 2 bits per symbol to 4 phase points:

● 00 → 45°
● 01 → 135°
● 11 → 225°
● 10 → 315°
Noise Level
OVERVIEW
QPSK carries 2 bits per symbol using 4 distinct phase shifts (45°, 135°, 225°, 315°).

It doubles the spectral efficiency of BPSK — same bandwidth, twice the data rate.

Gray Coding: Adjacent symbols differ by only 1 bit, reducing BER.
BER: Q(√(2·Eb/N0)) — same as BPSK per bit!

OVERVIEW
QAM modulates both amplitude and phase simultaneously, combining ASK and PSK.

16-QAM: 4 bits/symbol, 16 constellation points. Used in Wi-Fi, LTE.
64-QAM: 6 bits/symbol, 64 points. Higher data rate but needs clean signal.
256-QAM: 8 bits/symbol — used in cable TV & Wi-Fi 6.

Trade-off: Higher QAM order → more bits per symbol → more sensitive to noise.

OVERVIEW
Parity Check: Simplest method. Adds 1 bit to make total 1s even or odd. Detects single-bit errors only.

Hamming Code: Can detect up to 2-bit errors and correct 1-bit errors. Parity bits placed at power-of-2 positions.

CRC: Most powerful. Treats bits as polynomial coefficients and divides by generator polynomial. Remainder is the checksum. Widely used in Ethernet, USB, ZIP files.

OVERVIEW
TDM (Time Division Multiplexing): Multiple channels share the same medium by taking turns in time slots. Each channel gets a fixed slot in a repeating frame.

FDM (Frequency Division Multiplexing): Channels share the medium simultaneously but use different frequency bands. Guard bands prevent interference.

OFDM (Orthogonal FDM) is used in Wi-Fi and LTE — subcarriers are spaced so they don't interfere.

OVERVIEW
AWGN (Additive White Gaussian Noise) is the standard channel model in communications.

• "Additive" — noise adds to the signal
• "White" — equal power at all frequencies
• "Gaussian" — amplitude follows a normal distribution

SNR (Signal-to-Noise Ratio): Ratio of signal power to noise power, usually in dB.
SNR_dB = 10·log₁₀(P_signal / P_noise)

BER (Bit Error Rate): Fraction of bits received incorrectly.
For BPSK: BER = Q(√(2·Eb/N0))
